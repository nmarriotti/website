Error Handling is essential to ensure a script does not crash while running. In this tutorial I'll create a script with an error and show you how to implement error handling to prevent our script from crashing.


def main():
    x = 1/0
    print(x)

    print("Hi there")

Here we are trying to divide by zero. You can't divide by zero so when the script is executed we are immediately greeted with the below error and our script terminates.

Traceback (most recent call last):
  File "main.py", line 6, in 
    main()
  File "main.py", line 2, in main
    x = 1/0
ZeroDivisionError: integer division or modulo by zero

So how can we ignore this error?

Using try/except

We could try to exceute x = 1/0 and if it results in an error use the except statement to ignore and pass on that code. When looping you can also use the continue statement.


def main():
    try:
        x = 1/0
    	print(x)
    except ZeroDivisionError:
        continue

    print("Hi there")

The example above will only handle a ZeroDivisionError. If we want to catch all possible errors just use except.


def main():
    try:
        x = 1/0
    	print(x)
    except ZeroDivisionError:
        continue

    print("Hi there")

When we run our program now the error is ignored and our script finishes executing.
